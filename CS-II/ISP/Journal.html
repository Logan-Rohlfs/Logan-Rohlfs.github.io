1/2/23

Team Role

Software Engineer -

The software engineer is in charge of the main code base of the project and implementing features. They will also listen to the QA engineers and implement bug fixes when an issue it found. They are the backboard of any code based product. I think I am well suited for this role as I am very good at taking instruction and making sure the actual work gets done on time, I also want to work on my coding skills and this is the best opportunity to do so.



QA Engineer -

QA Engineer stands for Quality Assurance Engineer. This means that they are in charge of bug testing and performance testing a product to make sure the user experience is as intended. They work closely with the Product Managers and Software Engineers to assure the project is going as planned and all bugs get fixed. Just like the previous role I see this as an opportunity to better my coding skills and in addition being able to identify and fix bugs.



Project Manager -

The project manager is in charge of the bit picture concepts of the project and they organize the different sub-roles to make sure they are on task. They do less on the software side and more on the organization and plannign side to make sure the entire project is going as planned and in a timely manner. While I have good leadership experienc from football and have 100% confidence in being able to boss people around, I feel like this would be a detriment to my CS Education as I would not be coding as much as I would like to.



Product Manager -

The product manager works with the Software and QA Engineers to make sure the product is acceptable and without bugs and issues. They do some of the coding but mainly keep the Engineers in line. They are second in command from a management standpoint and work closer to the actual Software than the overall Project Manager. While the product manager does more coding and actual work on the project, I dont see myself being good at this role as I would spend a majority of my time managing and less of it doing the actual code.



Release Engineer -

Release Engineers are in charge of planning and organizing the releases of the product and make sure that all bugs and issues are fixed before any code goes live. They will either fix issues themselves or delegate back to QA and Software Engineers, they are the last line before code makes it out to the general public and hold the responsibility of keeping the code consistant. I think I would be very suitable for this role as it means I would not only be able to work on the actual code like a Software Engineer, I would also work on identifying and fixing bugs leading to a more rounded CS Education.
